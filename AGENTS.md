# Skills — Agent Rules

This file is the first-stop instruction set for any agent working in this repository.

## Mandatory Read Order

Before planning, editing, or reporting completion, every agent must read:

1. [README.md](./README.md) (if exists)
2. Project structure and purpose (see below)
3. The latest implementation notes (check for phase documents)

## Project Type

**Skills** is a **skills directory or tracking system** — likely managing:

- Skill categorization and taxonomy
- User/team skill profiles
- Skill assessments or tracking
- Skill marketplace or discovery

## Development Workflow

1. Review README.md and any existing documentation
2. Understand the current skill categorization structure
3. Make changes according to the project's conventions
4. Test and document changes
5. Commit with clear messages

## Completion Standard

An agent must not mark work complete until:

- All changes are applied and tested
- Documentation is updated
- Changes are committed with clear messages
- The final report distinguishes completed work, deferred work, and pre-existing issues

## Support

Review the project's README and documentation for specific details about structure and conventions.
